# Автономный Деплой на GitHub и Render (EduSecureShare)

Это краткое, пошаговое руководство, которое поможет вам выкачать проект, загрузить его в свой **GitHub** репозиторий и подключить к облаку **Render** для полностью автономной работы в интернете 24/7 без запуска на локальном компьютере через VS Code!

---

## ШАГ 1: Экспорт и сохранение проекта
1. На левой или верхней панели инструментов **Google AI Studio** откройте меню настроек проекта (иконка шестеренки или меню экспорта).
2. Выберите **Export to ZIP** (Экспортировать как архив) или клонируйте репозиторий напрямую, если подключена интеграция с GitHub.
3. Распакуйте архив на своем компьютере в любую рабочую папку.

---

## ШАГ 2: Создание репозитория на GitHub
1. Перейдите на сайт **[github.com](https://github.com)** и войдите в свой аккаунт.
2. Нажмите кнопку **New** (Создать новый репозиторий).
3. Назовите репозиторий (например, `edu-secure-share`). Сделайте его **Public** (публичным) или **Private** (приватным).
4. Оставьте флажки создания README, .gitignore и лицензий **пустыми** (невыбранными). Нажмите кнопку **Create repository**.
5. На своем компьютере в папке проекта запустите терминал и выполните следующие команды:
   ```bash
   git init
   git add .
   git commit -m "Initial commit for university project"
   git branch -M main
   # Замените ссылку ниже на URL вашего созданного репозитория:
   git remote add origin https://github.com/ВАШ_НИКНЕЙМ/edu-secure-share.git
   git push -u origin main
   ```

---

## ШАГ 3: Настройка и деплой на Render.com
Render — это популярный бесплатный облачный хостинг, поддерживающий Node.js.

1. Перейдите на сайт **[render.com](https://render.com)** и зарегистрируйтесь через свой GitHub аккаунт.
2. В панели управления Render нажмите кнопку **New +** в правом верхнем углу и выберите **Web Service**.
3. Выберите **Build and deploy from a Git repository** и подключите ваш репозиторий `edu-secure-share` из GitHub.
4. Введите следующие настройки для вашего веб-сервиса:
   - **Name:** `edu-secure-share` (или любое удобное имя для ссылки).
   - **Language:** `Node` (Node.js).
   - **Branch:** `main` (или ветка, в которую вы залили код).
   - **Root Directory:** Оставьте пустым (корневая папка).
   - **Build Command:** 
     ```bash
     npm install && npm run build
     ```
     *(Эта команда установит все пакеты, соберет клиентскую часть React 19 через Vite во внешнюю статику и скомпилирует бэкенд в единый файл CJS).*
   - **Start Command:**
     ```bash
     npm run start
     ```
     *(Эта команда запустит готовый оптимизированный сервер, выполняющий команду `node dist/server.cjs` на порту 3000).*

## ШАГ 4: Подключение API Ключа Gemini ИИ (Очень Важно!)
Ваше приложение использует искусственный интеллект Gemini для ИИ-Консультанта. Чтобы ИИ работал автономно в облаке:

1. В настройках вашего Web Service на Render перейдите в раздел **Environment** (Переменные окружения).
2. Нажмите кнопку **Add Environment Variable** (Добавить переменную окружения).
3. Введите:
   - **Key (Ключ):** `GEMINI_API_KEY`
   - **Value (Значение):** Вставьте ваш секретный API-ключ Gemini, полученный в Google AI Studio.
4. Нажмите **Save changes** (Сохранить изменения).

---

## ШАГ 5: Готово к использованию!
* Render автоматически начнет сборку (`npm run build`). Она занимает около 1-2 минут.
* После успешной сборки статус изменится на **Live** (В сети).
* В левом верхнем углу панели Render появится прямая ссылка на ваш готовый работающий сайт (вид: `https://edu-secure-share.onrender.com`).
* Теперь ваш проект работает полностью автономно в глобальном интернете — его можно открывать с любого телефона или компьютера, отправлять однокурсникам и преподавателям для демонстрации!
